# Canonical 2026-06-29 Amendment — Direct Nine-Fragment Concord Model

Status: this amendment supersedes any earlier sections in this document that describe The Concord as requiring six generic technology pieces, Ancient Components, or separate Echo Crystal / Meaning Lens / Concord Core gameplay gates. Those names may remain internal lore or visual subassembly labels only. Current canonical design is: nine island-specific Concord fragments collected through the existing 3x3 technology collection foundation complete The Concord and unlock translated inhabitant/creature communication in a later implementation PR.

- Island: Luma Isle.
- Scope: expedition technology.
- Construction: nine physical Concord fragments mapped one-to-one to slots 0-8.
- Collection UI: existing 3x3 technology collection modal.
- Completion: all nine distinct slots collected; no separate general-component requirement.
- Permanent unlock: translated communication with inhabitants and creatures, gated by canonical gameplay state in future implementation.

When the ninth fragment is collected, future implementation should reveal the final image section, illuminate the complete Concord image, play the completion celebration, name The Concord, explain the communication ability, and only then allow the communication system to become available through canonical services. The current 3x3 persistence and reward system remains the gameplay foundation and is not retuned by this documentation.

# The Concord — Universal Communication Technology

Status: Product/story concept and implementation investigation brief.

Scope:

- Island Run technology progression,
- inhabitant communication,
- creature communication,
- retro conversation presentation,

## 1. Core concept

The player’s first major technology is a universal communication device known formally as The Concord.

The player begins Island Run unable to fully understand the inhabitants and creatures of the islands.

Early communication is limited to:

- gestures,
- symbols,
- emotional reactions,
- broken or distorted words,
- creature sounds,
- visual thought-icons,
- basic responses such as nodding, pointing, or showing peaceful intent.

During Island 1, the player discovers and assembles the nine physical Concord fragments needed to restore The Concord.

Once activated, it becomes the in-world explanation for:

- talking with island caretakers and inhabitants,
- communicating with creatures,
- translating any language,
- interpreting non-verbal communication,
- showing conversations through the existing retro RPG interface.

The retro UI is therefore not merely a visual skin.

It is the device’s translation display.

## 3. Why communication appears in a retro interface

The Concord does not translate words alone.

It translates meaning into a form the player can safely and clearly understand.

It can interpret:

- speech,
- unfamiliar languages,
- creature calls,
- gestures,
- posture,
- magical pulses,
- scents,
- images,
- memories,
- emotion,
- instinct.

Directly receiving all of this would be overwhelming.

The Concord simplifies communication into:

- a compact retro scene,
- one speaker at a time,
- short dialogue pages,
- simple response choices,
- symbolic portraits,
- clear translated meaning.

Canonical lore explanation:

> The Concord does not translate words alone. It translates meaning into a form your mind can safely understand.

## 7. Creature communication rules

The Concord gives the player understanding.

It does not create automatic friendship.

It may translate communication such as:

> The creature sends an image of a warm nest beneath the trees.

Or:

> The creature’s clicking rhythm becomes a thought:  
> “The red stones frighten the hatchlings.”

The player must still:

- earn trust,
- respond respectfully,
- help solve problems,
- understand personality,
- build a bond,
- return later,
- bring the right companion,
- account for cultural differences.

The device translates communication, not relationships.

## 9. Relationship to the existing inhabitant system

The current two-stage inhabitant system remains the intended presentation:

1. premium inhabitant encounter,
2. player selects a topic,
3. retro conversation mode,
4. return to encounter or board.

The Concord changes when this communication becomes available.

Before The Concord is completed:

- no unrestricted translated inhabitant conversations,
- no full creature conversations,
- only limited non-verbal or partial communication where needed.

After activation:

- Talk to Caretaker becomes fully available,
- retro translated dialogue becomes available,
- creature communication can be introduced,
- future islands can reuse the same system.

The Concord must unlock the existing communication system.

It must not become a second dialogue engine.

## 14. Architectural principles

- Gameplay writes must remain in canonical Island Run action services.
- Technology definitions should describe content, requirements, channels, and asset references.
- Technology content must not contain arbitrary callbacks.
- The technology system must not duplicate conversation content.
- The inhabitant flow remains presentation-only.
- The Concord’s canonical activation state gates communication entry points.
- Component grants and activation must be idempotent.
- Existing per-island technology collection should be reused where appropriate.
- Do not build a large generic crafting engine for the first implementation.
- The Concord becomes globally permanent after activation.
- Island 1 is the acquisition story; later islands use the unlocked device.

## 15. MVP definition

The first implementation should include:

- one technology definition: The Concord,
- Island 1 acquisition story,
- three named components,
- reuse of the current technology-piece mechanic,
- a technology progress/build modal,
- one activation celebration,
- gating of Talk to Caretaker,
- first translated Caretaker conversation,
- Creature Channel shown as available or newly detected,
- no Island 2+ rollout yet.

Deferred:

- large Workshop catalogue,
- all future technologies,
- inter-island communication,
- full conversation archive,
- AI-generated dialogue,
- generic crafting recipes,
- broad creature-dialogue rollout,
- automatic story behavior outside Island 1.
